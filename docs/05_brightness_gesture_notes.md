# 밝기 조절 제스처 구현 정리
기존 ON/OFF 손 제스처 인식 프로젝트(`gesture.py`, `gesture_demo.py`)에 핀치(엄지-검지 거리) 기반 밝기 조절 모드를 추가

## 1. 목표
- 최종적으로는 실제 조명 하드웨어의 밝기를 손 제스처로 조절하는 것이 목표 
- 아직 하드웨어가 없으므로, 우선 핀치 거리로 0~100% 밝기 값을 계산하고 화면에 시각화하는 것까지 구현
    - 시각화 항목: 현재 제스처 이름("밝기조절") 텍스트, 계산된 밝기 값(%) 채워지는 만큼 채워지는 진행 바(bar), 캘리브레이션용 원본 핀치 거리(dist) 디버그 텍스트
- 하드웨어가 생기면 값 계산/인식 로직은 그대로 두고, 값을 적용하느 함수 하나만 교체할 수 있도록 설계

## 2. 동작 방식
1. 진입: 엄지+검지만 펴고 나머지 세 손가락(중지/약지/소지)은 접은 자세를 취하면 밝기 조절 모드로 진입
2. 유지: 한 번 진입하면 중지/약지/소지가 다시 펴지기 전까지 모드 유지 (핀치를 오므릴 때 엄지/검지 펴짐 판정이 흔들려도 모드에서 빠지지 않음)
3. 조절: 모드 중에는 엄지-검지 끝 사이의 실제 거리(`hand_world_landmarks`, 미터 단위)를 0~100%로 매핑해서 밝기 값 계산
4. 해제: 중지/약지/소지 중 하나라도 펴지면 모드 해제, 이후 ON/OFF 등 기존 제스처 인식으로 복귀 

### 밝기 모드 사용 흐름은 항상 다음 순서로 가야 함:
1. 엄지검지를 벌린 상태로 만들어서 밝기 모드 진입 
2. 그 다음부터 오므리고 벌리면서 밝기 조절
**처음부터 엄지-검지가 붙어있는 채로는 모드에 못 들어감**  
(붙이는 동작 자체가 "펴짐" 판정을 흔들어서 진입 조건을 통과하지 못함) -> 반드시 벌린 상태에서 시작

### world landmarks를 쓰는 이유
MediaPipe의 일반 `hand_landmarks`는 화면(이미지) 기준 정규화 좌표라 손이 카메라에 가깝고 멀고에 따라 같은 손 모양이어도 좌표 간 거리가 달라짐. `hand_world_landmarks`는 손 크기를 기준으로 환산한 실제 3D 좌표(미터)라 카메라와의 거리 변화에 영향을 덜 받으므로 핀치 거리 측정에는 world landmarks를 사용함.

### 진입/유지 조건을 분리한 이유
처음엔 "엄지+검지 펴짐, 나머지 접힘" 패턴이 유지되는 동안만 밝기 모드로 판단하려 했으나, 실제로 엄지-검지를 붙이는 동작을 하면 그 판정 자체(펴짐/접힘)가 흔들려서 모드가 끊기는 문제가 있었음.

또한 "중지/약지/소지가 접혀 있으면 무조건 밝기 모드"로 단순화하면, 완전한 주먹(OFF 제스처)도 같은 조건을 만족해서 OFF와 구분이 안 되는 문제가 생김.

최종적으로 **진입은 엄격한 패턴으로, 유지는 느슨한 조건으로, 해제는 별도 조건**으로 나누는 상태 머신 방식으로 해결:
- 진입: `classify_gesture`가 정확히 "밝기조절" 패턴과 일치할 때만
- 유지: 중지/약지/소지가 계속 접혀 있는 동안 (엄지/검지 상태는 무시)
- 해제: 중지/약지/소지 중 하나라도 펴질 때

이렇게 하면 처음부터 주먹을 쥔 경우(밝기 모드를 거치지 않은 경우)는 진입 조건 자체를 만족 못해서 정상적으로 OFF로 인식됨.

## 3. 캘리브레이션 결과 
`pinch_to_brightness`의 기본값(`min_dist=0.02`, `max_dist=0.10`)을 실측으로 검증:

| 손 모양 | 측정된 dist | 계산된 밝기 |
|---|---|---|
| 완전히 벌림 | 0.1115 | 100% (max_dist 초과 → 클램프) |
| 최대한 오므림 | 0.0237 | 5% |

가장 오므렸을 때도 완전히 0%까지는 안 내려가고 3\~5% 선에서 바닥을 형성함. OFF 제스처가 완전 꺼짐을 담당하므로 밝기 슬라이더가 굳이 0%까지 닿을 필요는 없다고 판단, 기본값 그대로 사용하기로 결정. (0%까지 닿게 하고 싶으면 `min_dist`를 0.022\~0.023 정도로 낮추면 됨.)

## 4. 트러블슈팅
- 핀치 중 펴짐 판정 흔들림: 엄지-검지를 실제로 붙이면 get_finger_states의 펴짐/접힘 판정이 흔들려서 모드가 끊기는 문제 → 상태 머신(진입/유지/해제 조건 분리)으로 해결
- OFF와의 충돌: 중지/약지/소지 접힘만으로 밝기 모드를 판단하면 완전한 주먹(OFF)도 같은 조건이라 혼동 → 진입 조건을 "엄지+검지가 펴진 정식 패턴"으로 엄격하게 제한해서 해결

## 5. 확장 지점 (하드웨어 연결 시)
`apply_brightness(value)` 함수 내부만 교체하면 됨. 나머지 제스처 인식, 상태 머신, 거리 계산 로직은 그대로 유지
